# フェーズ 5: review

## ゴール
成果物を自己レビューし、問題なければ承認を記録する。

## やること
1. plan の受入基準を 1 件ずつ照合する。
2. 各ソースファイルが 200 行以下か、禁止語が無いか、責務が分かれているかを再確認する。
3. 問題がなければ承認を報告する:
   ```
   harness report-gate review '{"verdict":"approved","notes":"<根拠の要点>"}'
   ```
4. 問題があれば前フェーズへ戻る:
   ```
   harness back "<どのフェーズの何が不足か>"
   ```
5. 承認を報告したら `harness advance` で完了。

## 禁止
- フェーズスキップ。
- `state/*.jsonl` の直接編集。
- 問題があるのに `verdict: approved` と報告すること。
