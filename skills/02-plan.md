# フェーズ 2: plan

## ゴール
research を踏まえ、実装計画を ≤200 行で書き切る。

## やること
1. `plan` という成果物（Markdown 推奨）を作り、最低限以下を含める:
   - 変更/新規ファイル一覧
   - 各ファイルの責務（1〜2 行ずつ）
   - テスト方針（どのコマンドで何を確認するか）
   - 受入基準（満たすべき条件の箇条書き）
2. 200 行を超えないこと。超えるなら責務を分割して計画も分ける。
3. ファイルを登録する:
   ```
   harness record-artifact plan <path>
   ```
4. 終わったら `harness advance`。却下されたら理由（plan_artifact_exists / plan_artifact_size_ok）を見て直す。

## 禁止
- フェーズスキップ。
- `state/*.jsonl` の直接編集。
- 成果物に禁止語（TODO, TBD, WIP, FIXME, 未定, 未確定, 要検討, 検討中, 対応予定, サンプル, ダミー, 仮置き）を残すこと。
