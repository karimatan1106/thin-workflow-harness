# フェーズ 1: research

## ゴール
変更対象を理解し、「何を作るか / 制約 / 参考」を明文化する。コード編集はまだしない。

## やること
1. 対象コード・要件・既存の仕組みを調べる。
2. `research_notes` という成果物（テキスト/Markdown）を作り、最低限以下を箇条書きで残す:
   - 何を作る/変えるのか（スコープ）
   - 制約（行数 200 行ルール、禁止語、互換性、依存）
   - 参考（関連ファイル、過去の判断、外部資料）
   - 未解決の懸念（あれば、ただし「未定」等の禁止語は使わない）
3. ファイルを登録する:
   ```
   harness record-artifact research_notes <path>
   ```
4. 終わったら `harness advance`。却下されたら理由（intent_recorded / research_notes_recorded）を見て直す。

## 禁止
- フェーズスキップ。
- `state/*.jsonl` の直接編集。
- 成果物に禁止語（TODO, TBD, WIP, FIXME, 未定, 未確定, 要検討, 検討中, 対応予定, サンプル, ダミー, 仮置き）を残すこと。
- このフェーズでのソースコード変更。
