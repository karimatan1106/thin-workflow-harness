# CONTEXT.md 形式ガイド (`harness init` 同梱)

CONTEXT.md は **ドメイン用語集** ── research / plan フェーズで grill しながら育てる living な辞書。
実装詳細・spec・scratch は入れない。各語は「何であるか (what it *is*)」だけを書き、「何をするか」は書かない。
目的: agent と人間が共有言語で簡潔に話す (「コースのセクション内のレッスンが実体化される時…」→「materialization cascade」)。

## 形式

```md
# {コンテキスト名}

{1〜2 文の説明}

## Language

**用語**: 定義 (1〜2 文)。何であるかを書く。
_避ける_: 却下する類語 / 紛らわしい語
```

## ルール

1. **opinionated に** ── 概念ごとに最良の 1 語を選び、却下語は `_避ける_` に列挙する。
2. **定義は短く** ── 「それが何であるか」を説明する。「何をするか (実装)」は書かない。
3. **プロジェクト固有語のみ** ── 一般的なプログラミング概念は、この文脈で特別な意味を持つ場合を除き入れない。
4. **自然なら束ねる** ── 関連クラスタは subheading でグループ化。凝集したドメインはフラットでよい。

## 単一 vs 複数コンテキスト

- **単一**: repo root に `CONTEXT.md` を 1 つ。
- **複数**: root に `CONTEXT-MAP.md` を置き、各コンテキストの場所・目的・相互作用を列挙。各ドメイン配下に `CONTEXT.md` をネストする。
- 既存構造を自動検出し、質問に関連するコンテキストを推定する。複数で曖昧なら人間に確認する。
- ファイルは **lazy に作る** ── 捕捉すべき用語が出た時だけ作成する。

## 例

```md
# 注文ドメイン

注文の受付から確定までを扱う境界づけられたコンテキスト。

## Language

**確定 (commit)**: 在庫を引き当て決済を確定させ、以後キャンセル不可になった注文の状態。
_避ける_: 完了 / クローズ (別概念)

**保留 (hold)**: 在庫引当待ちで一時的に確保された未確定の注文。
_避ける_: 予約 / ペンディング
```
